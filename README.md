# @aios-alpha/design

The one design system for every AIOS surface ‚Äî website, team brain, workspace GUI, and every scaffolded workspace. **Editorial Minimal** direction: dual-mode, greyscale chrome, Instrument typography, colour rationed to badges and data viz.

| Package | What it ships |
|---------|---------------|
| **`@aios-alpha/design`** (this repo root) | DTCG tokens ‚Üí `dist/tokens.css`, `dist/tailwind-theme.css`, `DESIGN.md` |
| **`@aios-alpha/ui`** (`react/`) | shadcn-based React components themed through the token bridge |

- **`DESIGN.md`** ‚Äî pinned design contract (like `brain-api.md` for sync). Read this before building UI.
- **`tokens/*.json`** ‚Äî DTCG source of truth. Edit here, then rebuild.
- **`dist/`** ‚Äî generated by `npm run build:tokens` (gitignored; CI rebuilds before publish).

## Prerequisites

- **Node** ‚Äî for the token + React build below.
- **[Git LFS](https://git-lfs.com)** ‚Äî the Pencil design source (`aios-design.pen`, a ~2 MB binary) is tracked via Git LFS, **not** plain git. Install it **once per machine before cloning** so the file materializes instead of leaving a small text pointer:

  ```bash
  git lfs install                                       # once per machine
  git clone git@github.com:AIOS-alpha/aios-design.git
  ```

  Already cloned before installing LFS? Run `git lfs install && git lfs pull`. See [`PENCIL.md`](./PENCIL.md) for opening the file.

## Build

```bash
npm install
npm run build:tokens   # regenerates dist/

cd react && npm install && npm run build   # compiles @aios-alpha/ui to react/dist/
```

## Consume in another repo

Install both packages (npm, same version ‚Äî currently **0.2.0**):

```bash
npm install @aios-alpha/design@^0.2.0 @aios-alpha/ui@^0.2.0
```

**Tailwind v4 app** ‚Äî global stylesheet, **in this order**:

```css
@import "@aios-alpha/design/tokens.css";
@import "@aios-alpha/design/tailwind-theme.css";
@import "tailwindcss";
@source "../node_modules/@aios-alpha/ui/dist";
@custom-variant dark (&:where(.dark, .dark *));
```

Self-host fonts (no Google CDN):

```css
@import "@fontsource/instrument-serif/400.css";
@import "@fontsource/instrument-sans/400.css";
@import "@fontsource/instrument-sans/500.css";
@import "@fontsource/instrument-sans/600.css";
@import "@fontsource/instrument-sans/700.css";
@import "@fontsource/jetbrains-mono/400.css";
```

Then use bridge utilities (`bg-background`, `text-foreground`, `bg-primary`, `text-violet`, `font-display`, `rounded-full`, ‚Ä¶). Toggle dark mode with `class="dark"` on `<html>`.

**React components:**

```tsx
import { Button, TierBadge, AiosMark, KpiStat } from "@aios-alpha/ui";
```

**Astro / Starlight** ‚Äî import `tokens.css` and map `--aios-*` onto Starlight's `--sl-*` (see `aios-website`).

**Raw CSS (no Tailwind)** ‚Äî import `tokens.css` and use `var(--aios-*)` directly.

### Semantic tokens (0.2.0)

| Token | Use for |
|-------|---------|
| `--aios-primary` | Pill buttons, active tabs (near-black light / white dark) |
| `--aios-violet` | Brand mark, team/deliverable badges, link accents |
| `--aios-accent` | Lime ‚Äî live status, checks, rationed affirmative actions |
| `--aios-emerald/amber/cyan/fuchsia` | Kind/tier badges, KPI sparklines |

### Migrating from 0.1.x

- **Fonts:** Space Grotesk + Plus Jakarta Sans ‚Üí Instrument Serif + Instrument Sans.
- **Buttons:** violet glow pills ‚Üí near-black/white editorial pills (`primary` token changed).
- **Badges:** team/deliverable now use `violet` token, not `primary`.
- **Local CSS bridges:** if you alias `--accent` to `--aios-primary`, split ‚Äî buttons stay on `primary`, link/highlight washes move to `--aios-violet`.

## Light vs dark

Light is `:root`. `class="dark"` on `<html>` swaps color values; type, spacing, and radius are mode-independent.
